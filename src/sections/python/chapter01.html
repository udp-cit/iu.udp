  1. ¿Qué es y por qué usar Python?

    Python es un lenguaje de programación de alto nivel, interpretado y de propósito general, conocido por su simplicidad y legibilidad. Fue creado por Guido van Rossum en 1991 y se utiliza ampliamente en diversas áreas como desarrollo web, análisis de datos, inteligencia artificial, automatización y más.

    Lenguaje interpretado

    Python es un lenguaje interpretado, lo que significa que su código no necesita ser compilado antes de ejecutarse. En lugar de traducirse a lenguaje máquina de inmediato, se ejecuta a través de un intérprete, lo que lo hace más flexible y portátil. Sin embargo, esto puede hacer que sea más lento en comparación con lenguajes compilados. Python también genera archivos .pyc o .pyo (bytecode) para optimizar la ejecución en futuras ocasiones.

    Tipado dinámico

    Python no requiere que especifiques el tipo de dato de una variable al declararla. En su lugar, el tipo se asigna en tiempo de ejecución según el valor que se le asigne. Esto permite mayor flexibilidad, pero también puede llevar a errores si el tipo de dato cambia inesperadamente.

    Fuertemente tipado

    Aunque Python permite el tipado dinámico, no permite operar directamente entre diferentes tipos de datos sin conversión explícita. Por ejemplo, si intentas sumar "9" + 8, obtendrás un error, ya que "9" es una cadena y 8 es un número. Para operar con ellos, tendrías que hacer una conversión, como int("9") + 8.

    Multiplataforma

    El intérprete de Python está disponible en una gran variedad de sistemas operativos, incluyendo Windows, macOS, Linux y otros. Mientras no se usen librerías específicas de una plataforma en particular, el código de Python puede ejecutarse sin cambios en cualquier sistema.

    Orientado a objetos

    Python sigue el paradigma de programación orientada a objetos, lo que significa que permite organizar el código en clases y objetos para representar conceptos del mundo real. Esto facilita la reutilización de código y la modularidad. Además, Python combina otros paradigmas como la programación imperativa y la funcional, lo que le da gran versatilidad.
